# 00_READ_FIRST — Constitutional Entry Point

**Status**: Active Governance  
**Version**: 1.0.0  

---

## 1. Constitutional Layer Purpose

The `PROJECT_CONSTITUTION/` directory is the supreme, authoritative context layer of the **AV Media Telangana Broadcast Kit** repository. It defines architectural intent, frozen module boundaries, governance rules, and implementation standards.

Every human contributor, AI model, Codex session, Gemini session, Claude session, or future engineering contributor MUST start here.

---

## 2. Mandatory Reading Order

All contributors and AI models MUST read the constitutional documents in the following strict order:

1. **`01_PROJECT_CONSTITUTION.md`** — Supreme governing principles and immutable laws.
2. **`02_PROJECT_STATE.md`** — Current active state and operational milestone status.
3. **`03_AI_DEVELOPMENT_RULES.md`** — Mandatory AI agent behavioral rules and coding constraints.
4. **`04_ARCHITECTURE.md`** — High-level system architecture, profiles, and data flow topologies.
5. **Remaining Constitutional Files (`05_SDK_REFERENCE.md` through `15_AI_QUICK_CONTEXT.md`)** — Read as required by specific engineering tasks.

---

## 3. Mandatory Rule

> **NO ENGINEERING WORK MAY BEGIN BEFORE THIS READING ORDER IS COMPLETED.**

No code edits, refactoring, design proposals, or file creations are permitted until the contributor or AI agent has fully read and acknowledged the primary constitutional documents (01 through 04).

---

## 4. Constitutional Authority

- **PROJECT_CONSTITUTION** defines **INTENT** and is the single authoritative context source.
- **Source Code** is the **IMPLEMENTATION**.
- **Chat Conversations** are **DISCUSSION HISTORY**.

In any conflict between discussion history, code, or documentation, the **PROJECT_CONSTITUTION** takes supreme precedence.

---

## 5. Synchronization Rule

> **EVERY COMPLETED ENGINEERING TASK MUST SYNCHRONIZE `PROJECT_CONSTITUTION` BEFORE IT IS CONSIDERED COMPLETE.**

No pull request, feature commit, or sprint task is considered complete until all relevant files in `PROJECT_CONSTITUTION/` (state, roadmap, ADR index, frozen modules) are updated to match the code implementation. Code and Constitution must NEVER diverge.

---

## 6. AI Startup Rule

Any AI coding agent (Gemini, Claude, GPT/Codex) MUST read and parse the `PROJECT_CONSTITUTION/` directory at the start of every session BEFORE suggesting architecture changes, writing code, refactoring existing logic, modifying documentation, or updating roadmaps.

---

## 7. Human Startup Rule

All human developers, open-source contributors, and broadcast engineers working on this repository must follow the exact same reading order and constitutional rules as AI agents.
